How To Make Translucent Models

Needed Files:
studiomdl.exe (for compiling)
The Transparent DLLs (so you can see the effects ingame)

This is a long read, but this is how you do it. This is how I do it, this is how everyone does it. If you want to learn (PLEASE WANT TO LEARN!!!), then you have to read it all. A retard can follow this tutorial, so if you can't, please kill yourself. :)

Since I'm getting like 1 trillion messages a day asking to make something transluscent, I'm making a simple tutorial. Before you begin, you need milkshape, the new studiomdl, and the new dlls. The dlls go into your halflife root folder. For me, that is D:/Halflife/.

If you are doing a default model, you will need to extract it from the pak file. You can do that with pakscape.

Step 1: For this tutorial I'm going to do the single shotgun. Copy the model into some crazy directory on your desktop (or somewhere else on your HD. Make sure it's in there by itself though).

Step 2: Copy studiomdl.exe into the folder you created.

Step 3: Open up milkshape. Go to tools->halflife->decompile normal hl mdl file and browse to the model you want to translucify. Make sure all 4 check boxes are checked.

Step 4: Go to the folder where you put your model at. Delete the model.

Step 5: Open up the v_tfc_12gauge.qc file in notepad. It should look something like this:

$modelname "C:\New Folder\v_tfc_12gauge.mdl"
$cd "\New Folder\"
$cdtexture "\New Folder\"
$cliptotextures

$scale 1.0

// 1 attachments
$attachment 0 "Bip01 R Hand" 30.0 1.0 6.0

// 0 bone controllers

// 5 hit boxes
$hbox 0 "Bip01 L Arm2" 0.0 -2.03 -2.94 12.71 1.84 2.2
$hbox 0 "Bip01 L Hand" 0.0 -0.79 -1.77 4.12 1.25 1.89
$hbox 0 "Bip01 L Finger0" -0.12 -1.22 -0.32 1.45 0.66 0.84
$hbox 0 "Bip01 R Hand" -5.19 -0.57 -0.59 24.37 1.46 5.79
$hbox 0 "Bone01" -0.54 -1.84 -1.9 6.9 1.03 1.79

$bodygroup studio
{
studio "v_tfc_12gauge_reference_REMAP"
}

$bodygroup studio
{
studio "shell"
}

// 10 sequences
$sequence sm_idle "sm_idle" loop fps 9
$sequence shoot "shoot" fps 20 { event 5001 0 "31" }
$sequence shoot_big "shoot_big" fps 13 { event 5001 0 "41" }
$sequence reload "reload" fps 16
$sequence pump "pump" fps 13
$sequence start_reload "start_reload" fps 12
$sequence draw "draw" fps 24
$sequence reholster "reholster" fps 30
$sequence idle4 "idle4" loop fps 9
$sequence deepidle "deepidle" loop fps 12

Now that you have your model decompiled, DON'T MOVE ANYTHING AROUND ON YOUR HARD DRIVE! It will mess up the paths at the beginning of the qc file and mess you up when you try to compile.

Step 6: Add this line into the qc file for every texture in the folder that you want to make translucent.

$texrendermode "texture_name.bmp" "additive"

For the single shotgun, you would add these lines.

$texrendermode "12gauge_chrome.bmp" "additive"
$texrendermode "12gauge_pump.bmp" "additive"
$texrendermode "12gauge_receiver.bmp" "additive"
$texrendermode "12gauge_stock.bmp" "additive"
$texrendermode "dm_base.bmp" "additive"
$texrendermode "rubberglovechrome.bmp" "additive"
$texrendermode "shell.bmp" "additive"

SPELLING IS VERY VERY IMPORTANT!!! If you make one mistake, it won't compile. Make sure you have the names EXACTLY like they are listed in the folder. I usually put these lines above the sequences part in the qc file. I don't know why, you can really put them anywhere.

A completed qc file would look like this:

$modelname "C:\New Folder\v_tfc_12gauge.mdl"
$cd "\New Folder\"
$cdtexture "\New Folder\"
$cliptotextures

$scale 1.0

// 1 attachments
$attachment 0 "Bip01 R Hand" 30.0 1.0 6.0

// 0 bone controllers

// 5 hit boxes
$hbox 0 "Bip01 L Arm2" 0.00 -2.03 -2.94 12.71 1.84 2.2
$hbox 0 "Bip01 L Hand" 0.0 -0.79 -1.77 4.12 1.25 1.89
$hbox 0 "Bip01 L Finger0" -0.12 -1.22 -0.32 1.45 0.66 0.84
$hbox 0 "Bip01 R Hand" -5.19 -0.57 -0.59 24.37 1.46 5.79
$hbox 0 "Bone01" -0.54 -1.84 -1.9 6.9 1.03 1.79

$bodygroup studio
{
studio "v_tfc_12gauge_reference_REMAP"
}

$bodygroup studio
{
studio "shell"
}

$texrendermode "12gauge_chrome.bmp" "additive"
$texrendermode "12gauge_pump.bmp" "additive"
$texrendermode "12gauge_receiver.bmp" "additive"
$texrendermode "12gauge_stock.bmp" "additive"
$texrendermode "dm_base.bmp" "additive"
$texrendermode "rubberglovechrome.bmp" "additive"
$texrendermode "shell.bmp" "additive"

// 10 sequences
$sequence sm_idle "sm_idle" loop fps 9
$sequence shoot "shoot" fps 20 { event 5001 0 "31" }
$sequence shoot_big "shoot_big" fps 13 { event 5001 0 "41" }
$sequence reload "reload" fps 16
$sequence pump "pump" fps 13
$sequence start_reload "start_reload" fps 12
$sequence draw "draw" fps 24
$sequence reholster "reholster" fps 30
$sequence idle4 "idle4" loop fps 9
$sequence deepidle "deepidle" loop fps 12

Step 7: Now that you have your completed qc ready to go, all you have to do is drag it onto the studiomdl.exe. If all goes well, it should compile and you should have a model there again. Copy the model into your halflife/tfc/models directory. For me, that would be D:/Halflife/tfc/models/. If you have the dlls in your halflife root like I said to do above, it should work fine.

Step 8: Pat yourself on the back, because you saved a customizer much time, and you learned something new, which might lead you to making other cool custom tfc stuff.

Just for Nam...

Step 9: Close out of milkshape before you run halflife, otherwise it will be choppy.
